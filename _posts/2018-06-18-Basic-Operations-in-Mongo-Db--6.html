Aggregation

There are scenarios where you want to display total scores of a student in a particular subject whose exam he/she has taken multiple times and calculate average for projection or number of goals a footballer has scored in a league in every season. For that we make use of aggregation.

Distinct and Count

> db.footballersData.distinct("fName") [ "Gareth", "Cristiano" ] > db.footballersData.count({fName:"Cristiano"}) 10

If we want to get the values for property i.e. if we have fName property in the documents and we want to know what are the values for that property in all documents where that property exists than we can make use of distinct method.

If we want to get the count of documents where a key value pair exists than we can make use of count method. If we want to get the count of all total number of documents in the collection than we can use count without passing it any filtering criteria.

Indexing

Whenever we work on a large application and size of our data grows than the time taken to fetch the documents also increases which is not liked by anyone, at that time indexes comes into picture. In my opinion index is just an array which holds the documents in sorted order (ascending or descending) and whenever a collection is queried using the key involved in index creation than our index array is scanned instead of the collection. Let’s take a look at it diagrammatically. Suppose we have an index on the league field in our collection in ascending order than it can be represented as

Property Value

Documents

Champions League

Doc1, doc2, doc3…..

La Liga

Doc1, doc2, doc3…..

Suppose we fire the following command to fetch the documents from our collection

db.footballersData.find ({league : “Champions League”})

Than instead of going to the collection and scanning all the documents in the collection to fetch the documents that meet our query criteria it will scan the index and will retrieve the documents.


By default _id field is the index that exists on our collection. To get the indexes created on a collection you can use getIndexes method on the collection.

> db.footballersData.getIndexes() [ { "v" : 2, "key" : { "_id" : 1 }, "name" : "_id_", "ns" : "myFirstDatabase.footballersData" } ]

In order to create an index on a collection we use createIndex method and then specify the properties that will participate in index creation and the order. Let’s take a look at creating an index:
